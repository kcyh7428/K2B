Which interaction pattern?

Click-to-expand works on both desktop and mobile (tap)

A

Click-to-expand everywhere

Consistent accordion pattern across all panels. Clean default, detail on demand. Tap-friendly on mobile.

B

Hover on desktop, tap on mobile

Hover tooltips for quick preview on desktop. Falls back to tap-to-expand on touch devices. More polished but two code paths.

C

Always show 1-line preview, expand for full

Every item gets a subtitle line by default (no click needed). Click/tap still expands for actions and full detail. More info at a glance but denser.
